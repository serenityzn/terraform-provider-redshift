# Redshift Provider

The Redshift provider provides configuration management resources for
[AWS Redshift](https://aws.amazon.com/redshift/), including both provisioned clusters and Redshift Serverless.

## Example Usage

### Authentication using fixed password

```terraform
provider "redshift" {
  host     = var.redshift_host
  username = var.redshift_user
  password = var.redshift_password
}
```

### Authentication using temporary credentials (provisioned cluster)

```terraform
provider "redshift" {
  host     = var.redshift_host
  username = var.redshift_user
  temporary_credentials {
    cluster_identifier = "my-cluster"
  }
}
```

### Authentication using temporary credentials in cross-account scenario (provisioned cluster)

```terraform
provider "redshift" {
  host     = var.redshift_host
  username = var.redshift_user
  temporary_credentials {
    cluster_identifier = "my-cluster"
    assume_role {
      arn = "arn:aws:iam::012345678901:role/role-name-with-path"
    }
  }
}
```

### Authentication using temporary credentials (Redshift Serverless)

```terraform
provider "redshift" {
  host     = var.redshift_host
  database = var.redshift_database
  type     = "serverless"
  temporary_credentials_serverless {
    workgroup_name = "my-workgroup"
    region         = "us-east-1"
  }
}
```

### Authentication using temporary credentials in cross-account scenario (Redshift Serverless)

```terraform
provider "redshift" {
  host     = var.redshift_host
  database = var.redshift_database
  type     = "serverless"
  temporary_credentials_serverless {
    workgroup_name = "my-workgroup"
    region         = "us-east-1"
    assume_role {
      arn = "arn:aws:iam::012345678901:role/role-name-with-path"
    }
  }
}
```

## Redshift Serverless — IAM Temporary Credentials

When using `temporary_credentials_serverless`, the provider calls `redshift-serverless:GetCredentials`
to obtain short-lived credentials from AWS. No password is stored in Terraform state.

The database username is derived from your IAM identity automatically:

| IAM identity | Redshift username |
|---|---|
| `arn:aws:iam::123456789012:role/my-role` | `iamr:my-role` |
| `arn:aws:iam::123456789012:user/john` | `iam:john` |

~> **Note:** The `username` field is ignored when using `temporary_credentials_serverless`.

### Limitations

The `redshift_user` resource does not work with `temporary_credentials_serverless`.
Redshift Serverless restricts `CREATE USER` for IAM-authenticated sessions, even for superusers.

**Workaround:** Use two provider instances — one with IAM credentials for schemas/grants,
and one with static admin credentials for user management:

```terraform
provider "redshift" {
  alias    = "iam"
  host     = var.redshift_host
  database = var.redshift_database
  type     = "serverless"
  temporary_credentials_serverless {
    workgroup_name = var.workgroup_name
    region         = var.region
  }
}

provider "redshift" {
  alias    = "superuser"
  host     = var.redshift_host
  database = var.redshift_database
  type     = "serverless"
  username = var.admin_username
  password = var.admin_password
}

resource "redshift_user" "example" {
  provider = redshift.superuser
  name     = "appuser"
  password = "StrongPassword123!"
}

resource "redshift_schema" "example" {
  provider = redshift.iam
  name     = "myschema"
}
```

## Proxy Support

If your Redshift cluster is only accessible from within a VPC, you can use the `ALL_PROXY` (`all_proxy`)
and `NO_PROXY` (`no_proxy`) environment variables to configure the provider to access Redshift through
a SOCKS5 proxy

```
ALL_PROXY=socks5[h]://[<socks5-username>:<socks5-password>@]<socks5-hostname>[:<socks5-port>]
NO_PROXY=#....
```

### `ALL_PROXY` URL parameters
* scheme: can be either `socks5` or `socks5h`. The `h` is optional and [does not change the behavior](https://cs.opensource.google/go/x/net/+/3a7c4785:proxy/proxy.go;l=92).
* `socks5-username` - optional username for authenticating to the socks proxy
* `socks5-password` - optional password for authenticating to the socks proxy
* `socks5-hostname` - the socks proxy hostname or IP address
* `socks5-port` - optional port number for the socks proxy. Default port is [1080](https://cs.opensource.google/go/x/net/+/3a7c4785:proxy/proxy.go;l=96)

### `NO_PROXY` Format

`NO_PROXY`/`no_proxy` is an optional environment variable which bypasses `ALL_PROXY`/`all_proxy`
for specific hosts. This is a [comma-separated string, where each value is one of](https://cs.opensource.google/go/x/net/+/3a7c4785:proxy/per_host.go;l=92-96):
* IP address (e.g. `127.0.0.1`)
* CIDR range (e.g. `192.168.0.0/24`)
* zone (e.g. `*.example.com`)
* hostname (e.g. `localhost`)
